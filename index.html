How PassAudit protects your privacy
01

Local hashing. Your password is converted to a SHA-1 hash entirely inside your browser using the Web Crypto API. It never leaves your device in plain text.

02

k-Anonymity model. Only the first 5 characters of your 40-character hash are sent to the Have I Been Pwned API — making it mathematically impossible to reverse-engineer your password.

03

Local comparison. The API returns thousands of matching hash fragments. Your browser checks them locally — the server never learns your full hash.

04

Instant audit. You see exactly how many times that password appeared in real-world data breaches — and a full strength analysis so you know exactly where you stand.
